# Commit Policy

## 기본값

사용자가 코드나 문서 변경을 요청했고 작업이 완료되면, 별도 지시가 없어도 커밋까지 진행한다.

## 커밋하지 않는 경우

다음 경우에는 커밋하지 않는다.

- 사용자가 `커밋하지 마`, `검토만`, `초안만`, `논의만`처럼 커밋하지 말라고 명시한 경우
- 작업이 실패했거나 검증할 수 없는 상태로 남은 경우
- 변경 범위에 사용자의 unrelated 변경이 섞여 있어 안전하게 분리할 수 없는 경우

## 커밋 전 확인

커밋 전에는 다음을 확인한다.

- `git status --short`
- 변경 diff
- 요청 범위에 포함되는 파일과 포함되지 않는 파일
- 실행한 검증과 실행하지 못한 검증

## Stage 규칙

- 요청 범위에 직접 관련된 파일만 stage한다.
- 사용자가 만든 것으로 보이는 unrelated 변경은 stage하지 않는다.
- Unity 생성 파일, 임시 파일, 실험 산출물은 요청 범위에 필요한 경우에만 stage한다.

## 보고

커밋 후에는 다음을 보고한다.

- 커밋 해시
- 변경 요약
- 검증 결과
- 검증하지 못한 항목이 있으면 그 이유
