# Documentation Policy

## 기본 원칙

외부 원문, 설치된 skill, setup이 생성한 문서는 그 도구가 실제로 기대하는 의미와 경로를 보존한다.

로컬 프로젝트 정책과 충돌해 보이더라도, 원천 문서의 용어와 경로를 임의로 재해석하지 않는다. 먼저 원천 파일과 관련 skill 문서를 읽고, 실제 의존 관계를 확인한다.

## 금지 사항

- OpenAI, Matt skills, Karpathy guideline 같은 원천 문서의 의미를 로컬 정책에 맞춰 요약하거나 치환하지 않는다.
- setup이 만든 경로를 확인 없이 다른 로컬 경로의 alias로 바꾸지 않는다.
- tool이 사용하는 문서 경로를 "우리 프로젝트에서는 다른 의미"라고 덮어쓰지 않는다.
- 생성된 문서가 어색해 보여도, 해당 tool이 실제로 소비하는 문서인지 확인하기 전에는 정리하지 않는다.

## 충돌처럼 보일 때

충돌처럼 보이면 다음 순서로 처리한다.

1. 관련 skill 또는 원천 문서를 먼저 읽는다.
2. 해당 경로가 tool이 실제로 읽거나 생성하는 경로인지 확인한다.
3. 같은 이름의 개념이 로컬 정책에도 있다면 대체 관계인지, 별도 용도인지 구분한다.
4. 별도 용도라면 둘 다 유지하고 역할을 문서에 명확히 적는다.
5. 정말 대체해야 한다면 사용자 확인을 받은 뒤 변경한다.

## 현재 적용 예시

Matt skills의 `CONTEXT.md`와 `docs/adr/`는 skill이 실제로 사용하는 domain glossary와 ADR 경로다.

SVM의 `docs/decisions/`는 bootstrap/governance decision 기록용이다.

따라서 `docs/adr/`를 `docs/decisions/`로 해석하거나 치환하지 않는다. 두 문서 트리는 목적이 다르며, 필요하면 각각의 역할을 유지한다.

이 규칙을 만든 직접 계기는 `docs/agents/ai-mistakes.md`에 기록한다.
