# Prompt Result Template

이 파일을 복사해서 `docs/prompt-results/001-topic-name.md`처럼 새 결과 파일을 만들고, 대괄호 placeholder를 실제 내용으로 바꾼다. 해당 없는 항목은 비워두지 말고 `없음` 또는 `해당 없음`으로 적는다.

## 실험 번호

`[예: 001]`

## 실험 주제

`[예: LLM/Codex 구조 이해]`

## 실행 일시

`[예: 2026-05-21 10:20 KST]`

## 실행 모드

- [ ] 일반
- [ ] 튜터

## 사용한 프롬프트

```text
[Codex에게 실제로 보낸 프롬프트 원문을 붙여넣는다.]
```

## 실행 조건

- 파일 수정 허용 여부: `[예: 파일 수정 금지 / 결과 파일만 생성 허용 / 수정 허용]`
- 참고 문서: `[예: docs/prompts/001-llm-codex-structure.md / 없음]`
- 사용한 skill/tool: `[예: openai-docs / browser / 없음]`
- 별도 제약: `[예: Unity 파일 수정 금지, 구현 계획으로 넘어가지 않기]`

## Codex 응답 요약

`[핵심 답변을 3-5문장으로 요약한다. 긴 답변을 그대로 복사하지 않는다.]`

## 튜터 모드 상호작용 요약

일반 모드 실행이면 모든 항목에 `해당 없음`을 적는다. 튜터 모드 실행이면 전체 대화 전문을 붙여넣지 말고 핵심만 요약한다.

### Codex가 한 확인 질문

- `[시작 질문 또는 이해 확인 질문을 요약한다.]`
- `[예: AGENTS.md와 skill의 차이를 어떻게 이해하고 있는지 물었다.]`

### 사용자 답변 요약

- `[사용자가 답한 핵심 내용을 요약한다.]`
- `[예: AGENTS.md는 repo 규칙, skill은 반복 작업 절차로 이해한다고 답했다.]`

### teach-back 결과 또는 아직 헷갈린 지점

- `[사용자가 자기 말로 설명한 결과나, 아직 헷갈린 개념을 적는다.]`
- `[예: hook과 skill의 차이는 설명했지만 MCP와 plugin의 경계는 추가 설명이 필요했다.]`

### 다음 복습 질문

`[다음에 스스로 답해볼 질문 1개를 적는다.]`

## 내가 이해한 점

- `[새로 이해한 개념이나 구조를 적는다.]`
- `[예: AGENTS.md는 반복되는 repo 규칙을 자동으로 context에 넣는 용도다.]`

## 실제로 도움 된 점

- `[앞으로 프로젝트 운영에 바로 써먹을 수 있는 점을 적는다.]`
- `[예: skill은 반복 workflow에만 쓰고, 일회성 질문은 prompt로 남긴다.]`

## 헷갈린 점

- `[아직 불명확하거나 추가 확인이 필요한 점을 적는다.]`
- `[예: hook을 warning으로 둘지 blocking으로 둘지 기준이 부족하다.]`

## 위험하거나 과해 보인 점

- `[scope creep, 잘못된 자동화, 유지보수 부담, Unity 파일 손상 위험 등을 적는다.]`
- `[예: 너무 이른 MCP/plugin 도입은 실험 결과를 흐릴 수 있다.]`
