# Safe Village Micro

Safe Village Micro는 작은 마을의 하루 생존 선택을 플레이어가 직접 배치하고 결과를 확인하는 초소형 생존 운영 게임이다. 이 문서는 프로젝트에서 쓰는 domain language와 용어 경계를 정한다.

## Language

**2D 마을 보드 playable**:
플레이어가 마을, 장소, 주민, 위험을 한 화면에서 보고 주민 배치와 하루 결과를 공간적으로 이해할 수 있는 playable 형태.
_Avoid_: UI-only playable, 텍스트 버튼 검증물, 단순 폼 UI

**고정 한 화면 마을 보드**:
화면 전환 없이 하나의 마을 화면 안에서 장소, 주민, 자원 상태, 위험 상태를 함께 보여주는 보드 형태.
_Avoid_: 다중 화면 UI, 메뉴 중심 진행, 화면별 폼

**하루 생존 루프**:
하루 단위로 주민을 배치하고 낮 결과, 밤 위험, 생존 여부를 확인하는 반복 구조.
_Avoid_: day cycle, 턴 진행, 라운드

**주민 배치**:
제한된 주민을 그날의 행동이나 장소에 나누어 놓는 플레이어 선택.
_Avoid_: 입력값, 카운터 조정, 액션 선택

**클릭 선택 배치**:
플레이어가 주민 토큰을 클릭해 선택하고, 장소를 클릭해 그 주민을 배치하는 조작 방식.
_Avoid_: 드래그 앤 드롭, 숫자 증감 버튼, 폼 입력

**작업 장소**:
주민 배치를 공간적으로 드러내기 위해 마을 보드에 고정된 장소. 현재 작업 장소는 `Forage`, `Guard`, `Repair`를 각각 나타내는 세 곳이다.
_Avoid_: 메뉴 항목, 숫자 버튼, 새 행동 규칙

**주민 토큰**:
마을 보드에서 배치 가능한 주민 1명을 나타내는 동일한 말. 현재 주민 토큰은 이름, 능력, 성격 차이를 갖지 않는다.
_Avoid_: 캐릭터, 직업, 스킬 보유자

**밤 위험**:
하루가 끝날 때 마을을 위협하며 방어 상태를 시험하는 외부 위협.
_Avoid_: 랜덤 이벤트, 전투, 좀비 웨이브

**결과 피드백**:
주민 배치가 Food, Wall, 생존 여부에 어떤 영향을 줬는지 플레이어가 이해할 수 있게 보드에서 먼저 드러나는 반응. 로그는 결과 피드백의 보조 정보로만 둔다.
_Avoid_: 로그 중심 진행, 디버그 메시지

## Flagged Ambiguities

**게임처럼**:
이 프로젝트에서는 새 규칙을 늘리는 뜻이 아니라, 기존 하루 생존 루프를 2D 마을 보드 playable로 느껴지게 만드는 것을 뜻한다.

## Example Dialogue

Dev: "다음 개선에서 규칙을 추가할까요?"

Domain expert: "아니요. 지금은 하루 생존 루프는 유지하고, UI-only playable을 2D 마을 보드 playable로 바꾸고 싶어요."

Dev: "그러면 주민 배치는 카운터가 아니라 장소 위의 주민 토큰으로 보여주고, 결과 피드백은 로그보다 마을 상태 변화로 먼저 보이게 하겠습니다."

Domain expert: "보드는 여러 화면이 아니라 한 화면에서 마을과 선택 상태가 같이 보여야 해요."

Dev: "주민은 드래그하지 않고 클릭 선택 배치로 움직이면 되나요?"

Domain expert: "네. 주민을 클릭하고 장소를 클릭해서 배치하는 방식이면 충분해요."

Dev: "작업 장소는 새 규칙이 아니라 Forage, Guard, Repair를 화면에 공간화한 세 곳으로 보면 되나요?"

Domain expert: "맞아요. 숲이나 폐허, 마을 문이나 벽, 작업대나 보수 지점처럼 보이면 됩니다."

Dev: "주민은 이름이나 능력이 있는 캐릭터가 아니라 동일한 주민 토큰 세 개로 보면 되나요?"

Domain expert: "맞아요. 지금은 배치 가능한 말이면 충분합니다."

Dev: "하루 결과는 로그보다 보드 변화가 먼저 보여야 하나요?"

Domain expert: "맞아요. 로그는 남겨도 되지만, Food와 Wall 변화, 실패나 승리는 보드에서 먼저 보여야 합니다."
